# 実行プレイブック

## Purpose

この文書は、複雑なタスクを進めるときの共通プレイブックを短くまとめたものです。正本のルールは `AGENTS.md` に置き、この文書は実行時の手引きだけを扱います。

## When To Use A Checklist

- 失敗を繰り返している
- 論点が増え続けている
- 役割分担や検証順序を明示したい
- 仕様、実装、テスト、レビューをまとめて動かす必要がある

## New Feature Flow

1. 期待する振る舞いを 1 文で定義する
2. 可能なら先に失敗テストか受け入れテストを置く
3. 最小実装で通す
4. 差分、近接テスト、補助確認を残す
5. 必要な範囲だけ整理する

テスト駆動開発（TDD）を外す場合は、外す理由と代替検証を報告に含めます。

## Narrow Edit Flow

1. 隣接実装と対象テストを読む
2. 最小変更で直す
3. 最も近いテストを回す
4. 差分や検索で副作用が広がっていないことを確認する

## Collaboration Escalation

- 単独で進めるより、役割分担したほうが安全か速いと判断したときだけ協働を使う
- 協働を始める前に、対象パス、書き込み範囲、レビュー出力先、終了条件を固定する
- Codex と Model Context Protocol（MCP）を使った相談テンプレートが必要なら `docs/ai/checklists/codex-mcp-collaboration-template.md` を使う

## Design Notes

- 一時的なログや長い議事録は正本にせず、要点だけを正本 docs か設計判断メモに蒸留する
- ルールを増やす前に、既存の正本へ追記できないか確認する
