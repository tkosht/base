# Codex と Model Context Protocol（MCP）を使う協働チェックリスト

困難タスクの打開を目的に、別の AI エージェントと協働するための運用テンプレートです。推測禁止、事実ベース、セキュリティ厳守を前提にします。

## 0. 原則

- [ ] 推測ではなく、再現、ログ、差分で記述する
- [ ] 鍵、トークン、個人情報を出さない
- [ ] 外部調査は必要なときだけ行う
- [ ] 長大ログはそのまま貼らず、要点だけに絞る

## 1. エスカレーション条件

- [ ] 30 分以上停滞した
- [ ] 3 回以上試しても有効な前進がない
- [ ] 原因の切り分けが難しい
- [ ] 設計のトレードオフが拮抗している
- [ ] ローカルでは再現せず、継続的インテグレーション（CI）やレビューでだけ問題が出る

## 2. 準備する情報

- [ ] Problem: 現象と期待を 1〜3 文で書く
- [ ] Environment: 対象 repo、バージョン、依存を書く
- [ ] Reproduction: 最短手順と抜粋ログを書く
- [ ] Attempts: 試したことと結果を書く
- [ ] Hypotheses: 根拠付きの仮説を書く
- [ ] Constraints: 制約を書く
- [ ] Ask: 今回のゴールを 1 つに絞る
- [ ] Artifacts: 関連ファイル、行番号、テスト名、PR を列挙する

## 3. 協働中のルール

- [ ] 対象ファイルを明示して範囲を絞る
- [ ] 出力形式を指定する
- [ ] 重要な提案だけを手元で実行する
- [ ] 成否はログか差分で記録する

## 4. 収束後

- [ ] 解決策または次善策が明確になった
- [ ] 要点を正本 docs か設計判断メモに蒸留した
- [ ] 一時メモや長いログを正本に混ぜていない
